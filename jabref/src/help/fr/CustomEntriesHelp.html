Personnaliser les types d'entrées

[NdT: pour accéder à cette fonction, cliquez sur le menu Options/ Personnaliser les types d'entrées]

Lors de la personnalisation d'un type d'entrée, vous définissez à la fois son apparence dans l'éditeur d'entrées et ce qui est nécessaire pour que JabRef considère une entrée comme complète. Vous pouvez aussi bien changer les types d'entrée existants qu'en définir de nouveaux.

Utiliser la fenêtre de personnalisation des entrées

Dans la partie supérieure gauche de la fenêtre se trouve une liste où vous pouvez sélectionner un des types d'entrées actuellement définis. Toutes les entrées personnalisées, qu'elles correspondent à de nouveaux types ou à des types standards redéfinis, sont affichés avec un astérisque (*) à la fin de leurs noms.

Sélectionner un type d'entrées ne fait qu'afficher ses champs requis et optionnels, vous donnant un modèle pour la personnalisation. Vous pouvez alors changer à la fois la définition et le nom de ce type d'entrée avant de le sauver. Si le nom apparaissant dans le champ de texte Name [NdT: Quel champ Name ? Probablement le champ en bas à gauche de la fenêtre] est différent de celui du type sélectionné, le champ Name est prioritaire quand vous cliquez sur Ajouter ou Supprimer.

Vous pouvez supprimer un type d'entrées personnalisé en la sélectionnant et en cliquant sur le bouton Supprimer ; si la définition supprimée redéfinissait un type standard, la définition standard sera restaurée. Vous ne pouvez pas effacer les types d'entrées standards.

Noter qu'il n'y a pas de fonction d'annulation lorsque vous créez, modifiez ou supprimez des définitions de type d'entrées.

Quand vous sélectionnez un des types standards de la liste de types, la syntaxe définissant les champs requis et optionnels devrait vous apparaître évidente ; les noms de champs sont simplement écrits dans l'ordre que vous le souhaitez, séparés par des points-virgules. [NdT: ça n'a pas vraiment l'air de marcher comme ça] N'importe laquelle des listes de Champs requis et Champs optionnels peuvent être vide, s'il se trouve qu'aucun des champs ne correspondent à cette catégorie. Les champs listés n'ont pas à être les champs standards connus de JabRef.

Redéfinir les types d'entrées standards

Si vous sauvez la définition d'un type d'entrées avec un nom identique à celui d'un des types standards, le type personnalisé sera prioritaire sur le type standard. Vous pouvez utiliser cette propriété pour redéfinir l'ordre dans lequel les champs apparaîssent dans l'éditeur, ajouter de nouveaux champs que vous utilisez, supprimer des champs qui ne vous intéressent pas et même déplacer tous les champs dans la liste des Champs requis afin de ne pas avoir à changer d'onglet lorsque vous éditez des entrées. Il y a une limitation à la personnalisation des entrées ; certains types ont une condition et/ou dans les champs requis. Par exemple, une entrée book est complète lorsque au moins un des champs author et editor est rempli. Ce type de condition ne peut pas être introduite dans une entrée personnalisée. Si vous supprimez une version personnalisée d'un type d'entrée standard, la définition standard sera restaurée.
